# planning

**Fase de Planejamento Operacional**

O mapa de trabalho. Transforma os requisitos em um cronograma executavel com ciclos
curtos e entregas verificaveis.

## Para que serve
Quebra o projeto em Milestones, Sprints e tarefas atomicas. Define como o codigo sera
testado e validado antes de chegar ao usuario. Impede que o time comece a codar sem
saber o que vem primeiro.

## Quando executar
Apos a aprovacao dos requisitos. Antes da primeira linha de codigo.

## Beneficios
- Sprints curtas forcam entregas verificaveis a cada 5-7 dias
- O roadmap macro garante que o Sponsor veja o progresso sem precisar ler o codigo
- O plano de testes escrito agora impede que "testar" seja a ultima tarefa que nunca acontece
- A sequencia de priorizacao garante que o MVP exista antes dos detalhes

## Sequencia de Execucao
1. `roadmap.md` — Milestones macro com datas
2. `sprint-plan.md` — Tarefas atomicas com criterio de aceite por sprint
3. `testing-plan.md` — Estrategia de testes (unit, integracao, E2E)
4. `validation-plan.md` — Cenarios de validacao com usuario real

## Criterio para Avancar
Existe um "Corte de MVP" identificado no roadmap. O primeiro sprint tem tarefas com 
criterio de aceite definido.
